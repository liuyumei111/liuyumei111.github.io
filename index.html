基础node1

发表于 2017-02-27

1.服务端没有window对象，后台最大的是global对象，在文件中运行this不是global
2.通过var声明，不能直接声明在global上

global上的属性 (全局对象，直接访问)

  • process 进程对象
    • nextTick 下一队列（当前队列的底部）
    • process.pid 进程的ID
    • process.kill()杀死进程
    • process.cwd()当前工作目录（__dirname不可以更改的）
    • process.exit()退出进程
  • Buffer 缓存 16进制 二进制
  • setTimeout/clearTimeout
    setTimeout是异步，中的this指的是它自己，bind可以改变this指向
  • setInterval/clearInterval
  • setImmediate/clearImmediate
  • console
    - console.log/error/info/warn/time/timeEnd

文件目录与文件名

dirname(全局变量存储的是文件目录) filename（全局变量存储的是文件名）

异步

  • process.nextTick 同步代码执行后会执行的代码（第一个小本的尾部）
  • setImmediate 异步在nextTick后执行（第二个小本）
  • setTimeout 唯一能设置执行时间的 （第二个小本）

顺序：同步>process.nextTick>setImmediate>=settimeout

模块化的特点

  • cmd（seajs）旧近依赖 amd(requirejs)依赖前置
  • commonjs（node）读写文件I/o操作
  • 模块化有利于分工协作
  • 实现高内聚低耦合
  • 实现代码复用

闭包，命名空间（1.不能完全避免命名冲突2.调用过长）

如何实现模块化

  • 定义模块化（创建一个js文件就是定义了一个模块）
  • 使用模块（需要用谁require谁）
  • 导出模块（exports / module.exports）

文件模块

  • 引用方式要通过相对路径引用./开头
  • 引用多次 只执行一次
  • 导出模块（exports/module.exports）

第三方模块npm

  • npm是管理后台文件的 管理前端文件的是bower
  • 别人的模块 都在npm网站中

  • 下载文件都是通过npm

    • 全局下载-g，电脑上的所有项目都可以访问（只在命令下使用）

      • nrm工具（切换源-切换到国内下载东西网速快）

      • nodeppt基于node秒做ppt的插件

      • nodeppt安装npm install nodeppt -g

    • 本地下载（在代码里使用）下载到项目中目录中，本项目可用

      • 记录安装过的所有模块：

        • 在项目中新建一个文件夹（英文）
        • 进入当前目录命令行进行初始化文件夹npm init -y会自动生成一个package.json文件
        • 以后每次本地安装模块的时候在末尾添加–sava或者–save-dev 把安装模块基本信息会添加到package.json这个文件中，达到了记录安装了哪些模块

      • 卸载模块

查看模块/插件版本

不指定版本默认安装最新版本

第三方模块的查找

安装第三方模块会自动生成node_modules文件，第三方模块的查找查找当前目录下node_modules文件，找到名字相同的文件夹，找到package.json看里面的main，运行main对应的文件，找不到会向上级查找node_modules文件，直到找到为止（根目录），找不到则报错 （如果找不到 会默认找index.js）

bower (管理前端文件)

  • bower的操作和npm异曲同工

在npm官网上发布自己的包（一个包是由很多个模块（js）组成）

  • 找到带有package.json的文件夹(文件初始化出现package.json)
  • 切到官方npm上
